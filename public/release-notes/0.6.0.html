FinderGit 0.6.0

Release notes

Sidebar, redesigned

  • Virtual groups — Tower-style containers to organise your roots into Work, Side projects, Dotfiles, anything you want. Right-click any group to rename, delete, or move roots in and out.
  • Drag to reorder roots within the sidebar; positions persist across launches.
  • Inline filter at the top — type to narrow long sidebar lists. Matches both folder names and group names, so typing the name of a group surfaces every root inside it.

Detail panel, clearer

  • Changes list now splits into separate Staged and Unstaged sections, each with its own count and per-file Stage / Unstage.
  • The Git Only toggle and the file filter sit above the file browser column instead of floating in the window toolbar — visually anchored to the column they actually control.
  • Window resize no longer flips your detail layout from "right" to "bottom" silently. Your View → Detail at Bottom (⌘⌥D) preference is sticky.
  • Sidebar drag is bounded so it stops squeezing the detail panel off-screen at typical window widths.

Help that actually helps

  • ⌘? opens the live documentation at findergit.app/docs in your browser — always in sync with the version you're running.
  • A small What's New panel introduces every minor release on first launch, with a curated highlights list. Re-openable any time from the Help menu.

Polish

  • VoiceOver labels and ARIA semantics across the sidebar, the file browser, and the lightbox-able controls.
  • Fix: clicking a virtual group header no longer accidentally merges files from every root in that group into a mixed tree.
