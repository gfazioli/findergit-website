FinderGit 0.7.1

Release notes

FinderGit 0.7.1 — Swift 6 hygiene

Housekeeping release. Adds an @unknown default arm to the ValidationOutcome
switch in Settings → Git → GitHub Token that the Swift 6 language mode
would otherwise refuse to compile (the enum lives in a different module
than the switch, so Swift treats it as resilient).

No user-visible changes — the runtime behaviour for the token validation
flow is identical to 0.7.0. If you're already on 0.7.0 and everything is
working, there's no functional reason to update; this release exists to
keep the build clean ahead of the next Swift toolchain bump.

Under the hood

  • Added @unknown default arm to the switch await provider.validate() in
    GitSettingsView.saveAndValidateToken(). Future cases of
    GitHubAPIProvider.ValidationOutcome (e.g. a hypothetical .rateLimited)
    now map to a fail-closed outcome with a generic "Couldn't validate the
    token — try again" message, refusing to save a token whose validation
    we can't fully classify.
