FinderGit 0.11.0

Release notes

FinderGit 0.11.0 — Forks, branches, and a tidy working tree

Highlights

  • Fork awareness — GitHub forks now carry a badge showing how many commits they trail their upstream, with the parent repo one click away. When a fork is behind, Sync Fork fast-forwards it on GitHub and pulls your local clone — no more checking github.com repo by repo. Diverged forks are called out honestly instead of offering a sync that can't apply.
  • Branch switcher — a new Branches tab lists local and remote branches with one-click checkout (remote branches get a proper local tracking branch), plus a branch dropdown in the panel header and a Switch Branch submenu in the right-click menu. No more discovering mid-task you were on the wrong branch.
  • Discard changes — the Unstage tab gains a per-file Discard and a Discard All: modified files revert to the last committed version, untracked files go to the Trash (recoverable from Finder), always behind a confirmation.

Under the hood

  • Branch and ref names are validated before reaching git, closing an argument-injection edge case in the new checkout paths.
  • Background refreshes that return unchanged data no longer touch UI state — fixes the open branch menu visibly resizing a moment after opening.
  • The fork badge, branch list, and the other per-repo columns all refresh with the toolbar Refresh and stay cancellation-safe under rapid re-triggering.

Install

Download FinderGit-0.11.0.dmg below, open it, and drag FinderGit to Applications. Signed and notarized by Apple — no Gatekeeper bypass needed. Existing users update automatically in-app.

Requirements

macOS 15 (Sequoia) or later. Universal (Apple silicon + Intel).
