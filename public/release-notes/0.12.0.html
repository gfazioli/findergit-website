FinderGit 0.12.0

Release notes

Catch a supply-chain worm before it runs 🛡️

FinderGit 0.12.0 hardens Repo Trust against the Shai-Hulud / Miasma supply-chain worm — the family that hijacks a maintainer's account and commits an obfuscated setup.js dropper that runs the moment you open the repo in an editor, let an AI agent load it, or run npm install.

Repo Trust now catches the payload, not just the trigger

  • Dropper detection — FinderGit flags the obfuscated dropper itself (by known indicator path, or by its packed shape), not only the hooks that launch it.
  • Every branch, not just your checkout — the worm hides a back-dated copy on a side branch, invisible to a clean working tree. Repo Trust scans all refs with read-only git plumbing, so it can't slip past — and still never runs anything.
  • A wider net — dev-container lifecycle commands (which can run on your host) and always-applied AI-agent rules are surfaced too.

An incident runbook when a repo looks compromised

  • Rotate credentials the right way — revoke the whole OAuth authorization (a token rotation isn't enough — the grant stacks tokens under it), with direct links.
  • Clean the repo safely — copy-paste commands with your repo and the infected branches filled in: back up evidence, then reset (not revert — a revert leaves the payload reachable by its SHA).

Clearer failures, with the fix in hand

  • Git errors now read in plain language instead of a raw error dump — shown as a tidy icon + popover you can copy.
  • For known causes like a missing git-lfs, FinderGit explains why and hands you the one-line fix to copy.

A refreshed detail panel

  • Each tab has its own colour, and the Trust tab tints itself by risk — green when clean through to red when a dropper is present — so a problem stands out at a glance.
  • A new Buy me a coffee option in About & Support, alongside sponsorship.

Install

Download the DMG below, drag FinderGit into Applications, and launch it normally — the app is signed with Apple Developer ID and notarized by Apple, so Gatekeeper accepts it on first open. Already running FinderGit? Update in place from FinderGit → Check for Updates (or relaunch — the auto-updater will offer it).

Requirements

  • macOS 15 (Sequoia) or later
  • Xcode Command Line Tools for the system git (xcode-select --install)
  • Repositories that use Git LFS need git-lfs installed — FinderGit now detects when it's missing and shows the one-line fix

Repo Trust is read-only — FinderGit never executes, installs, or evaluates anything it inspects, so it's safe to point at a repo you don't trust yet.
