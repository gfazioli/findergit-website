FinderGit 0.10.1

Release notes

FinderGit 0.10.1 — Refresh you can see

Highlights

  • Refresh you can see — pressing Refresh now shows live progress on the Issues, PRs and Stars columns while it re-checks them, and refreshes every repository's on-disk size. A manual Refresh gives immediate, visible feedback instead of looking like nothing happened.

Bug fixes

  • The "Select an Item" placeholder arrow now points up toward the file list when the detail panel is docked at the bottom — it always pointed left before, regardless of layout.
  • Refresh now genuinely re-fetches the GitHub counts and repository sizes that were cached for the session, so the columns reflect what changed after a pull.

Under the hood

  • The toolbar Refresh drives a refreshAll() on the remote-counts and repo-size stores alongside the existing trust re-scan; the counts re-fetch bypasses the in-memory cache and re-enters the loading state so cells show their progress indicator.
  • The counts fetch is now fully cancellation-safe — a cancelled re-fetch (e.g. double-pressing Refresh) no longer writes terminal store state, which previously could pin a repo as "unsupported" and hide its counts.

Install

Download FinderGit-0.10.1.dmg below, open it, and drag FinderGit to Applications. Signed and notarized by Apple — no Gatekeeper bypass needed. Existing users update automatically in-app.

Requirements

macOS 15 (Sequoia) or later. Universal (Apple silicon + Intel).
